## 📌 Project Overview
This project performs a deep-dive exploratory data analysis (EDA) on vehicle market data to identify the hidden correlations between vehicle specifications and their market valuation. As a Data Scientist, my goal was to move beyond simple averages and uncover the non-linear relationships that drive pricing in the modern automotive secondary market.

## 🛠️ Technical Stack
* **Language:** Python 3.10
* **Analysis:** Pandas, NumPy
* **Visualization:** Matplotlib, Seaborn
* **Automation/DevOps:** GitHub Actions, Python Scripting
* **AI Integration:** Gemini 2.0/3 Flash API (Automated Quality Audit)

---

## 🤖 AI-Powered Data Governance (MLOps)
To maintain high standards of code integrity and analytical rigor, I built a custom **CI/CD pipeline** that acts as an automated "Senior Data Scientist" peer reviewer.

### How the Automation Works:
1.  **Trigger:** Every code push triggers a GitHub Action runner.
2.  **Audit:** A Python script (`auto_audit.py`) feeds the notebook logic to **Gemini 2.0 Flash**.
3.  **Validation:** The AI scans for logic flaws, statistical outliers, or missing data-cleaning steps.
4.  **Reporting:** A live audit report is generated automatically to ensure the project remains "production-ready."

## 📈 Key Insights & Methodology
* **Exploratory Data Analysis:** Investigated the distribution of price points and mileage across different vehicle classes.
* **Feature Engineering:** Analyzed the impact of "Year" vs. "Condition" on depreciation rates.
* **Statistical Correlation:** Identified which specific features (e.g., fuel type, transmission) act as the strongest predictors for market value.
